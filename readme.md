# Memora.AI — Your AI-powered study companion

Welcome to Memora.AI — a friendly, lightweight platform that helps you turn readings and PDFs into useful study material in minutes. Upload a PDF or start from a topic, generate a clear reviewer, create flashcards and quizzes, and keep your highlights organized in a personal memory bank.

---

**Core idea (casual):** Highlight what matters, let AI turn it into bite-sized study tools, and level up as you learn. Simple, fast, and made for real students.

---

**Key Features**

1. Create topics from a search/query or upload your own PDF — Memora will extract the text and build a topic entry.
2. Reviewer generation — produces a structured, multi-section study reviewer from your PDF or topic (generated by the app's AI service).
3. Supplementary AI — inside the reviewer you can add extra AI-generated supplements or tweak prompts for more focused content.
4. Highlighting panel — select words, phrases, or sentences in the reviewer; highlights are saved in a right-side panel for organization.
5. Memory Bank — everything you highlight becomes a `Memory` item and seeds quiz/flashcard generation.
6. Flashcards — generate sets of flashcards (mixed difficulties) from the reviewer and your highlights.
7. Quizzes — generate short/long/exam-style quizzes (multiple choice, identification, enumeration, modified true/false) based on the reviewer.
8. Gamification (experimental) — earn XP, level up, and collect badges when you complete quizzes and create content.

---

**What the codebase actually uses (technical)**

- Backend: FastAPI (Python) with a lightweight SQL database (SQLite by default). Frontend: React + Vite.
- Highlight & memory system: `Memory` model stores highlighted text, optional notes and color. These are used to bias flashcard and quiz generation.
- PDF handling: PyMuPDF (via `backend/services/pdf_service.py`) extracts text and images from uploaded PDFs.
- AI provider: the running code uses Groq via the OpenAI-compatible SDK (`backend/services/ai_service.py`) with model `llama-3.3-70b-versatile` (configured by `GROQ_API_KEY` in `backend/config.py`).

Note: the repository contains an older `.env.example` entry for `GEMINI_API_KEY`, but the active backend configuration expects `GROQ_API_KEY`. There is no automatic fallback to Gemini in the current code.

Quick start (local dev)

Prerequisites: Python 3.10+, Node.js 18+, Git.

Backend (API)

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r backend/requirements.txt
# Set environment vars (example):
setx GROQ_API_KEY "your_groq_api_key"
setx SUPABASE_URL "your_supabase_url"
setx SUPABASE_KEY "your_supabase_anon_key"
uvicorn backend.main:app --reload --port 8000
```

Frontend (UI)

```bash
cd frontend
npm install
npm run dev
```

API notes

- Upload PDFs at `POST /api/topics/upload` — the API extracts text and (optionally) generates a brief summary.
- Generate a full reviewer at `POST /api/topics/{topic_id}/generate-reviewer`.
- Create highlights via `POST /api/memories` and generate flashcards/quizzes using `/api/flashcards/generate` and `/api/quizzes/generate` (the server will include user highlights when calling the AI).

Contributing & customization

- If you want Gemini instead of Groq: the code would need the AI service rewritten to call Google Gemini APIs and the environment updated; currently there's no automatic fall-back path.
- To change the model, edit `backend/services/ai_service.py` and `backend/config.py`.

Privacy & limits

- Uploaded PDFs and generated content are stored in your backend database/uploads directory by default. Treat the instance as you would any web app — do not upload sensitive material to a public host.
- AI rate limiting is handled simplistically; heavy usage may hit provider quotas.
